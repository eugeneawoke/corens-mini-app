# Дизайн-система Corens MVP

Этот документ описывает базовую дизайн-систему для Telegram Mini App `corens` (MVP-версия), ориентированную на спокойное, теплое и интроспективное пространство.

## 1. Design principles
1. **Тишина в интерфейсе.** Минимум визуального шума. Интерфейс должен ощущаться как пустая, теплая комната.
2. **Цвет как сигнал.** 95% интерфейса монохромно-нейтральные. Цвет появляется только для обозначения состояния, действия или смыслового акцента.
3. **Бережность к пользователю.** Плавные формы, щедрые отступы и отсутствие агрессивных паттернов привлечения внимания.
4. **Приватность по умолчанию.** Закрытые данные (фото, контакты) визуально отделены и защищены до явного согласия.
5. **Ясность намерений.** Типографика и лейблы должны считываться мгновенно, без двойных смыслов.

## 2. Color system
Основа — светлая тема (Light Theme). Темная тема (Dark Theme) предполагается в будущем как инверсия с глубокими темно-серыми (не чисто черными) фонами.

- **background**: `#F9F9F8` (Теплый off-white)
- **surface**: `#FFFFFF` (Чистый белый для карточек)
- **surface-elevated**: `#FFFFFF` (С легкой, размытой тенью для модалок)
- **text-primary**: `#1C1C1E` (Почти черный, мягкий)
- **text-secondary**: `#8E8E93` (Нейтральный серый)
- **text-tertiary**: `#C7C7CC` (Для плейсхолдеров и неактивных элементов)
- **border-subtle**: `#E5E5EA` (Едва заметные разделители)
- **accent-primary**: `#6C6AC4` (Приглушенный индиго, основное действие)
- **accent-primary-hover**: `#5856A8` (Для tap states)
- **success**: `#3D9B58` (Успешное действие, подтверждение)
- **warning**: `#C4811A` (Ожидание, предупреждение)
- **danger**: `#C4453C` (Удаление аккаунта, отмена)
- **beacon-active**: `#9B6BC4` (Приглушенный фиолетовый для режима Beacon)
- **consent-approved**: `#3D9B58` (Зеленый индикатор)
- **consent-pending**: `#C4811A` (Оранжевый индикатор)
- **reveal-locked**: `#E5E5EA` (Серый блюр/заглушка)
- **reveal-open**: `#6C6AC4` (Цвет акцента при открытии)

## 3. Typography
- **Font Families:**
  - Заголовки: `Satoshi` (вес Medium и Bold)
  - Наборный текст: `Inter` (вес Regular и Medium)
  - Счетчики/Таймеры: `JetBrains Mono` (вес Regular)
- **Heading Scale:**
  - H1: 28px, Line Height 1.2 (Satoshi Bold) — Экраны приветствия
  - H2: 22px, Line Height 1.3 (Satoshi Medium) — Заголовки секций
  - H3: 18px, Line Height 1.4 (Satoshi Medium) — Заголовки карточек
- **Body Scale:**
  - Body 1 (Primary): 16px, Line Height 1.5 (Inter Regular) — Основной текст
  - Body 2 (Secondary): 14px, Line Height 1.4 (Inter Regular) — Подсказки, описания
- **Caption/Meta Scale:**
  - Caption: 12px, Line Height 1.3 (Inter Medium) — Теги, статусы, время
- **Button Text:** 16px (Inter Medium)
- **Mono Usage:** Только для таймеров (например, время работы Beacon). Размер 14px или 16px.

## 4. Spacing and sizing
- **Базовый шаг:** 4px / 8px.
- **Ключевые отступы:** 16px (внутренние отступы карточек, боковые поля экрана), 24px (между смысловыми блоками), 32px (отступы до кнопок действий).
- **Container width rules:** Вписывается в ширину экрана мобильного устройства (100%), боковые margins: 16px.
- **Safe area behavior:** Учитывать `safe-area-inset-bottom` и `safe-area-inset-top` от Telegram Mini App (особенно для Bottom Action Bar).
- **Радиусы (Border Radius):**
  - Кнопки, поля ввода: `12px`
  - Карточки, модалки: `20px`
  - Мелкие элементы (чипы, бейджи): `8px` или `100px` (pill shape).
- **Высоты контролов:**
  - Кнопки, текстовые поля: `48px` (оптимальный touch target).
  - Чипы: `32px`.

## 5. Core components only

**App Shell**
- *Purpose:* Основная обертка экранов.
- *Anatomy:* Фон `background`, опциональный Top Bar, контентная область (scrollable), зафиксированный Bottom Action Bar.

**Top Bar**
- *Purpose:* Навигация и контекст.
- *Anatomy:* Кнопка "Назад" (иконка), Заголовок (опционально, H3), кнопка доп. действия (справа).
- *Spacing:* Высота 44px + safe area.

**Section Header**
- *Purpose:* Разделение логических блоков.
- *Anatomy:* H2 (Satoshi), отступ снизу 12px.

**Primary Button**
- *Purpose:* Главное действие на экране.
- *Anatomy:* Фон `accent-primary`, текст `surface` (белый), радиус 12px. Высота 48px.
- *States:* Default, Active (opacity 0.8), Disabled (фон `#E5E5EA`, текст `#8E8E93`).

**Secondary Button**
- *Purpose:* Альтернативное действие.
- *Anatomy:* Фон `surface`, граница 1px `border-subtle`, текст `text-primary`. Высота 48px.

**Ghost Button**
- *Purpose:* Второстепенное или отменяющее действие (например, "Пропустить").
- *Anatomy:* Без фона, текст `text-secondary` или `danger` (для удаления). Высота 48px.

**List Item**
- *Purpose:* Отображение настроек, контактов.
- *Anatomy:* Иконка (опционально, слева), текст Body 1, стрелка или значение справа. Padding 12px 0 (с разделителем) или 16px 16px (внутри карточки).

**Card**
- *Purpose:* Группировка информации (Trust Keys, State).
- *Anatomy:* Фон `surface`, радиус 20px, padding 16px. Возможна легкая тень `0 2px 8px rgba(0,0,0,0.04)`.
- *Selected state:* Тень-свечение цветом акцента `box-shadow: 0 0 48px 4px rgba(108, 106, 196, 0.67)`. Обычная граница `border-subtle` сохраняется. Не использовать обводку `accent-primary` — только тень.

**Status Badge**
- *Purpose:* Индикация состояния (Pending, Approved).
- *Anatomy:* Pill shape (радиус 100px), фон — 10% от цвета статуса (например, зеленый или оранжевый), текст — 100% цвета статуса. Высота 24px, текст Caption.

**Chip / Selectable Tile**
- *Purpose:* Выбор State / Intent, Trust Keys.
- *Anatomy:* Бордер `border-subtle`, фон прозрачный. При выборе — бордер `accent-primary`, фон 5% `accent-primary`, иконка красится в акцент. Высота 40-48px (для плиток) или 32px (для мелких тегов).

**Bottom Action Bar**
- *Purpose:* Фиксация главного действия поверх скролла.
- *Anatomy:* Блок с градиентом от прозрачного к `background` снизу, чтобы текст не сливался, внутри Primary Button. Padding 16px + safe area.

**Modal / Bottom Sheet**
- *Purpose:* Фокус на одном решении (Privacy warning, Photo Reveal).
- *Anatomy:* Выезжает снизу. Фон `surface`, верхние углы радиус 24px. Оверлей черный 40% (или блюр). Внутри: иконка, H2, Body 1, кнопки действия.

**Form Field & Textarea**
- *Purpose:* Ввод текста.
- *Anatomy:* Фон `surface`, рамка 1px `border-subtle`. Текст Body 1. Высота поля 48px, текстареа — min 100px. При фокусе рамка `accent-primary`.

**Toggle / Switch**
- *Purpose:* Настройки приватности.
- *Anatomy:* Стандартный iOS-подобный свитч (включен — `success`, выключен — `border-subtle`).

**Consent Card & Privacy Warning Block**
- *Purpose:* Блок передачи контактов или предупреждения.
- *Anatomy:* Карточка с выделенным фоном (например, 5% `warning` для предупреждений). Иконка `warning` (Phosphor, outline). Текст Body 2.

**Empty State Block**
- *Purpose:* Нет связей, нет сообщений.
- *Anatomy:* Центрирование по экрану. Легкая абстрактная иконка/мотив (Cloud), заголовок H3, текст Body 2 (text-secondary).

## 6. Visual motifs
- **Cloud / Crystal / Beacon:** Использовать только как 2D-иллюстрации (SVG, размытые цветовые пятна или векторные line-art элементы с мягкими градиентами). 
- **Где уместно:** В Empty States, на экране приветствия, на карточке включения Beacon.
- **Где не уместно:** В списках, формах, системных диалогах.

## 7. Interaction guidelines
- **Tap states:** Для всех интерактивных элементов (карточек, кнопок, list items) — мгновенное снижение прозрачности до 60% (или наложение черного слоя с opacity 5%).
- **Transitions:** Мягкие появления (fade in 200ms) для контента. Модалки выезжают снизу (slide up 300ms, ease-out).
- **Haptics suggestions:** `Light` haptic при нажатии на чипы/теги. `Medium` при активации свитчей и главных кнопок. `Success` при мэтче или одобрении контакта.
- **Feedback:** При загрузке (сохранение формы) кнопка переходит в disabled state с индикатором загрузки (Spinner). Никаких блокирующих весь экран лоадеров без нужды.

## 8. Do / Don’t
1. **Do:** Использовать пространство (воздух) для разделения элементов вместо линий.
2. **Don’t:** Не использовать тени для создания иерархии (максимум — легчайшая тень для модалок, все остальное плоское). Исключение: тень-свечение `accent-primary` для обозначения выбранного состояния карточки.
3. **Do:** Использовать иконки Phosphor (Light или Regular) для сохранения чистоты и тонкости линий.
4. **Don’t:** Не красить тексты в яркие цвета, кроме статусов и ссылок (индиго).
5. **Do:** Показывать заблокированный контент (Photo Reveal) стильно — мягким блюром или серой плашкой с иконкой замка.
6. **Don’t:** Не использовать анимации ради анимаций. Никаких прыгающих кнопок.
7. **Do:** Делать кнопки отмены или отказа (Decline) визуально тихими (Ghost Button), чтобы не создавать стресс.
8. **Don’t:** Не перегружать экран выбора (State / Intent) пестрыми иконками — использовать строгий монохром, который красится при выборе.
9. **Do:** Четко отделять системные предупреждения (Privacy) рамками или мягким фоном.
10. **Don’t:** Не использовать кастомные шрифты для цифр, где важна моноширинность (таймеры Beacon) — только JetBrains Mono.
