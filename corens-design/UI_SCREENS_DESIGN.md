# Проектирование экранов Corens MVP

Этот документ описывает дизайн-реализацию экранов Telegram Mini App `corens` на основе утвержденной дизайн-системы.

## 1. Overall UI direction
Интерфейс `corens` — это минималистичное, функциональное пространство без лишнего декора. Он использует паттерны "настроек" или "утилит" (списки, карточки, простые формы) вместо развлекательных механик. Главный принцип — **спокойствие и контроль**. Пользователь всегда понимает, где он находится, что происходит с его данными, и какие действия от него ожидаются.

## 2. Navigation and app structure
- **Primary Screens:** Home (точка входа), Profile (настройки и стейт). Переключение между ними через нативный `BackButton` Telegram (когда мы в Profile, ведем на Home) или простым переходом.
- **Secondary Screens:** Connection, Beacon, State / Intent, Trust Keys, Privacy. Открываются либо пушем нового экрана (для сложных флоу), либо как `Modal / Bottom Sheet` (если нужно быстрое действие или решение).
- **Bottom Bars:** В Telegram Mini App основная навигация часто скрыта. Мы используем зафиксированный Bottom Action Bar (главная кнопка) только на экранах, требующих явного завершения (сохранение State, включение Beacon).
- **Связь с Telegram API:** Использовать `tg.MainButton` там, где это уместно, чтобы сэкономить место на экране (например, для сохранения профиля).

## 3. Screen-by-screen design

### Экран 1. Home
- **Goal:** Показать текущий статус (есть ли связи, включен ли Beacon) и дать точку входа в основное взаимодействие.
- **Information hierarchy:**
  1. Top Bar (справа иконка Profile).
  2. Main Status Block (по центру).
  3. Action Button (внизу).
- **Blocks:**
  - *Active Connection:* Карточка с базовой информацией о текущем мэтче. Нажатие ведет на экран Connection.
  - *No Match:* Empty State Block (облако/кристалл) + текст "Ожидание совпадений...".
  - *Beacon Status:* Если активен — небольшая плашка с таймером (JetBrains Mono).
- **Primary CTA:** Включить Beacon (если нет связи).
- **Secondary actions:** Перейти в Profile.
- **Empty states:** Центральное мягкое пятно (Cloud), спокойный текст "Сейчас у вас нет активных связей".

### Экран 2. Connection
- **Goal:** Детализация текущей связи, управление доверием.
- **Information hierarchy:**
  1. Profile Preview (размытое фото или плейсхолдер, имя/псевдоним).
  2. Совпавшие Trust Keys (список тегов).
  3. Статус этапа (State).
  4. Действия доверия.
- **Blocks:**
  - *Header:* Имя, статус.
  - *Keys Match:* Карточка с совпадениями.
  - *Trust Actions:* Блоки Contact Consent и Photo Reveal (показывают текущий статус: Locked, Pending, Approved).
- **Primary CTA:** Запросить фото / Запросить контакты (если доступно).
- **Secondary actions:** Отклонить / Разорвать связь (Ghost Button в самом низу).
- **Warning states:** Если связь разорвана — немедленный возврат на Home с нотификацией.

### Экран 3. State / Intent
- **Goal:** Настройка текущего эмоционального/жизненного состояния.
- **Information hierarchy:**
  1. Текущее состояние.
  2. Выбор нового.
  3. Ограничения (cooldown).
- **Blocks:**
  - Сетка `Selectable Tiles` (2 колонки) с простыми иконками Phosphor.
  - Текстовое описание выбранного состояния.
  - Cooldown Indicator (Caption, `text-tertiary`), если изменение заблокировано.
- **Primary CTA:** Сохранить (или `tg.MainButton`).
- **Telegram Mini App:** Удобно использовать нативную главную кнопку для сохранения.

### Экран 4. Trust Keys
- **Goal:** Выбор ценностей/ключей доверия.
- **Information hierarchy:**
  1. Счетчик (например, "Выбрано 3 из 5").
  2. Список ключей по категориям.
- **Blocks:**
  - Категории в виде Section Headers.
  - Группы `Chip` элементов. При нажатии чип красится в `accent-primary`.
- **Primary CTA:** Сохранить изменения.
- **Warning states:** Если выбрано больше лимита — чипы блокируются, легкий `Light` haptic.

### Экран 5. Beacon
- **Goal:** Активация режима активного поиска.
- **Information hierarchy:**
  1. Статус (Выключен / Работает).
  2. Время действия.
  3. Описание того, что произойдет.
- **Blocks:**
  - Иллюстрация (Beacon motif, мягкий градиент).
  - Таймер обратного отсчета (крупно, если активен).
  - Текстовый блок с объяснением приватности режима.
- **Primary CTA:** Активировать Beacon (в цвете `beacon-active`).
- **Secondary actions:** Деактивировать (Ghost Button).

### Экран 6. Profile
- **Goal:** Управление аккаунтом.
- **Information hierarchy:**
  1. Пользователь (имя, фото-заглушка).
  2. Ссылки на настройки (State, Keys).
  3. Приватность и данные.
- **Blocks:**
  - Список `List Item` (Намерение, Ключи доверия, Приватность).
  - Секция "Опасная зона" (Удаление аккаунта).
- **Primary CTA:** Нет. Навигационный экран.

### Экран 7. Contact Consent
- **Goal:** Обмен контактами (часто реализован как Bottom Sheet).
- **Information hierarchy:**
  1. Объяснение.
  2. Текущий статус партнера.
  3. Действие.
- **Blocks:**
  - *Consent Card:* Иконка замка/щита. Текст "Вы готовы обменяться контактами?".
  - Индикатор `consent-pending` или `consent-approved`.
- **Primary CTA:** Разрешить обмен (`success` цвет).
- **Secondary actions:** Отложить / Отказать.

### Экран 8. Photo Reveal
- **Goal:** Открытие фотографий (Bottom Sheet).
- **Information hierarchy:** Аналогично Contact Consent.
- **Blocks:**
  - Графический элемент: размытый квадрат, переходящий в фокус (схематично).
  - Текст "Показать фото?".
- **Primary CTA:** Открыть фото (`reveal-open` цвет).

### Экран 9. Privacy
- **Goal:** Настройки видимости.
- **Blocks:**
  - Список Toggle/Switch элементов (например, "Искать меня только по Beacon").
  - `Privacy Warning Block` с текстовым пояснением политик.

### Экран 10. Delete Account
- **Goal:** Подтверждение удаления.
- **Information hierarchy:** Максимально строго, без графики.
- **Blocks:**
  - Красный `Warning` блок. Текст: "Это действие нельзя отменить. Все связи и данные будут удалены."
  - Поле ввода (если нужно ввести "удалить" для подтверждения).
- **Primary CTA:** Удалить аккаунт (цвет `danger`).
- **Secondary actions:** Отмена (Primary Button или Ghost, чтобы увести от удаления).

## 4. Shared states
- **Loading:** Локальные спиннеры (например, в кнопках) вместо полноэкранных блокировок. Skeleton-экраны для загрузки Connection (серые прямоугольники с пульсацией opacity 0.3 -> 0.7).
- **Empty:** Элемент в центре экрана, иконка Phosphor (размер 48px, `text-tertiary`), заголовок H3, текст Body 2.
- **Success / Error:** Использование `tg.HapticFeedback`. Inline сообщения (Toast или текст под полем), а не popup alert.
- **Cooldown:** Отключенные кнопки (disabled state) с текстом "Доступно через 12:00" (JetBrains Mono).
- **Locked / Pending / Approved:** 
  - Locked: `reveal-locked` фон, иконка Lock.
  - Pending: `warning` (оранжевый) обводка или текст, иконка Clock.
  - Approved: `success` (зеленый) цвет, иконка Check.

## 5. Minimal visual accents
- **Cloud:** Мягкое круглое пятно (gradient off-white to #E5E5EA) на фоне Home при отсутствии связей.
- **Beacon:** Размытый `beacon-active` (#BF5AF2) круг с opacity 10% на экране Beacon.
- **Где чистый UI:** Формы, настройки, профиль — строго монохром, линии `border-subtle`, 0% декораций.

## 6. Implementation-minded notes
- **Что просто для TMA/Next.js:** Использование нативных компонентов Telegram (`MainButton`, `BackButton`, `HapticFeedback`).
- **Статичность первой версии:** Никаких сложных переходов между экранами. Обычный роутинг (React Router / Next.js Pages) без тяжелых анимаций выезда.
- **Optional polish:** Легкое появление (Fade In) контента на новых страницах.

## 7. Final constraint summary (10 правил)
1. Не добавлять кастомные навигационные бары снизу (Bottom Tab Bar) — используем меню и возврат.
2. Не проектировать интерфейс чата.
3. Не использовать 3D, видео или тяжелые Lottie/Rive анимации.
4. Ограничиться двумя начертаниями шрифта (Regular, Medium) для веба.
5. Не создавать сложных карточек с перекрывающимся контентом (строгий flow сверху вниз).
6. Никаких попапов, кроме нативных алертов или Bottom Sheets (для Consent).
7. Запрещено использовать тени для создания "слоев", кроме одного уровня (Surface Elevated для модалок).
8. Использовать только одну цветовую тему (Light) для первого релиза.
9. Иконки должны быть одного семейства (Phosphor) и одного веса.
10. Верстка должна быть строго одной колонкой (mobile-first), без адаптации под десктоп/планшет.
